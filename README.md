# 实验三：目录树查看器 —— 仿 Linux `tree` 命令（二叉树应用）

## 一、实验目的

**深入理解二叉树的链式存储与“左孩子右兄弟”表示法**：通过将多叉树（目录树）转化为二叉树，掌握二叉树结点结构的设计与递归创建方法。

**掌握二叉树的递归遍历算法**：实现先序遍历（用于输出树形结构），并理解递归在树形输出中的前缀处理技巧。

**熟练运用二叉树的统计操作**：计算结点总数、叶子结点数、树的高度，并理解这些指标在二叉树中的递归定义。

**培养递归思维与动态内存管理能力**：递归遍历文件系统目录，正确使用 `opendir` / `readdir` 等系统调用，并妥善处理动态分配内存的释放。

**通过实现一个实用的系统命令，提升综合编程能力**：将数据结构知识应用于实际场景，增强学习兴趣。

## 二、实验内容

设计一个 C 语言程序，实现一个简化版的 Linux `tree` 命令，能够递归显示指定目录下的所有文件和子目录，并以树形结构输出。程序需使用**左孩子右兄弟的二叉树**来存储目录树，并完成以下功能：

### 核心功能

1. **构建目录树**  
   - 根据用户输入的路径（默认当前目录），递归扫描目录。  
   - 为每个文件/子目录创建一个二叉树结点，结点包含：名称、类型（目录/文件）。  
   - 采用**左孩子右兄弟**结构组织：  
     - `firstChild` 指针指向第一个子项（长子）。  
     - `nextSibling` 指针指向下一个兄弟项（同目录下的其他条目）。  

2. **树形输出**  
   - 输出格式与 Linux `tree` 命令类似，使用 `|--`、`|   `、`` `-- `` 等前缀。  
   - 目录名后加 `/` 以示区分，文件不加。  
   - 输出整个目录结构，根目录名单独一行。  

3. **统计信息**  
   - 输出目录总数、文件总数。  
   - 输出二叉树的**结点总数**（每个文件和目录都是一个结点）。  
   - 输出**叶子结点数**（`firstChild == NULL` 的结点，即空目录或文件）。  
   - 输出**树的高度**（根深度为 1，空树高度为 0）。  

4. **内存管理**  
   - 程序结束前递归释放所有动态分配的结点内存，避免内存泄漏。

### 扩展功能（选做，加分）

- 支持 `-L` 参数限制显示深度（如 `./tree -L 2` 只显示两层）。  
- 支持按名称排序输出（默认按文件名升序）。  
- 支持输出重定向到文件（如 `./tree ~ > out.txt`）。

## 三、实验要求

### 1. 数据结构定义

- 二叉树结点（左孩子右兄弟）：
  ```c
  typedef struct FileNode {
      char *name;                  // 文件/目录名
      int isDir;                   // 1:目录 0:文件
      struct FileNode *firstChild; // 左孩子：第一个子项
      struct FileNode *nextSibling;// 右兄弟：下一个同层项
  } FileNode;
  ```

- 目录遍历使用 POSIX API：`opendir`、`readdir`、`closedir`、`stat`。

### 2. 基本操作要求

- **创建结点**：`FileNode* createNode(const char *name, int isDir)`
- **构建目录树**：`FileNode* buildTree(const char *path)`（递归实现）
- **树形输出**：`void printTree(FileNode *node, const char *prefix, int isLast)`
- **统计结点总数**：`int countNodes(FileNode *root)`
- **统计叶子结点数**：`int countLeaves(FileNode *root)`
- **计算树的高度**：`int treeHeight(FileNode *root)`
- **统计目录和文件数**：`void countDirFile(FileNode *root, int *dirs, int *files)`
- **释放整棵树**：`void freeTree(FileNode *root)`
- **获取当前目录名**：`char* getBaseName(void)`

### 3. 功能细节

**命令行参数**：
- 支持一个可选参数：`./tree [路径]`。  
- 若无参数，默认为当前工作目录。  
- 若路径不存在或不是目录，输出错误信息并退出。

**构建目录树**：
- 使用 `opendir` 打开目录，遍历每个条目，跳过 `.` 和 `..`。  
- 对每个条目，用 `stat` 判断类型：  
  - 目录：递归调用 `buildTree` 得到子结点。  
  - 普通文件：调用 `createNode` 创建文件结点。  
- 将得到的子结点按名称排序（可选，但推荐以便输出整洁），然后链接成兄弟链表（第一个挂在 `firstChild`，后续挂在 `nextSibling`）。

**树形输出**：
- 根目录名单独一行（不加前缀）。  
- 对根目录的每个孩子，调用 `printTree` 函数，传入合适的前缀和是否为最后一个孩子的标志。  
- `printTree` 函数需递归处理每个结点的孩子链表，根据深度动态构造前缀字符串（`"    "` 或 `"|   "`）。

**统计信息**：
- 结点总数 = 1 + 左子树结点数 + 右子树结点数。  
- 叶子结点数 = (firstChild == NULL ? 1 : 0) + 左子树叶子数 + 右子树叶子数。  
- 树高度 = max(1 + 左子树高度, 右子树高度)（空树返回 0）。  
- 目录/文件数：遍历整棵树累计。

**内存释放**：
- 递归释放左子树和右子树，最后释放当前结点的 `name` 和结点本身。

### 4. 代码规范

- 使用 C99 标准，文件名 `tree.c`。  
- 函数名、变量名要有意义，关键代码添加注释。  
- 每个函数职责单一，避免重复代码。  
- 输入输出格式清晰，错误信息应输出到 `stderr`。
